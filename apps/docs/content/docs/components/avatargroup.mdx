## Features

- Stacked (overlapping) or Grid layouts
- Overflow handling (`+3 more`)
- Hover effects for stacked items (expand on hover)
- Integrated with `Avatar` component for consistent styling
- Fully accessible `role="group"`

## Anatomy

```tsx
<AvatarGroup
  users={users}
  size="medium"
  maxDisplay={3}
  layout="stack"
  spacing="normal"
  onAvatarClick={(user, index) => console.log(user)}
  onOverflowClick={(hiddenUsers) => console.log(hiddenUsers)}
/>
```

| Element          | Description                                                           |
| :--------------- | :-------------------------------------------------------------------- |
| **Root**         | Container with `role="group"` for accessibility                       |
| **Avatar Items** | Individual avatars rendered from the `users` array                    |
| **Overflow**     | Indicator showing remaining count when users exceed `maxDisplay`      |

## Props

| Prop | Type | Default | Description |
|:-----|:-----|:--------|:------------|
| `users` | `UserData[]` | `[]` | Array of user objects |
| `size` | `'small' \| 'medium' \| 'large' \| 'xlarge'` | `'medium'` | Size of avatars |
| `maxDisplay` | `number` | `5` | Max visible before overflow |
| `layout` | `'stack' \| 'grid'` | `'stack'` | Layout mode |
| `spacing` | `'tight' \| 'normal' \| 'loose' \| number` | `'normal'` | Negative margin for stack |
| `onAvatarClick` | `(user: UserData, index: number) => void` | — | Click handler for avatars |
| `onOverflowClick` | `(hiddenUsers: UserData[]) => void` | — | Click handler for overflow button |

### UserData Interface

```tsx
interface UserData {
  id?: string | number | null;
  src?: string;
  username?: string;
  alt?: string;
  fallback?: string;
  status?: "online" | "offline" | "away" | "busy";
}
```

### Sizes

| Size     | Dimensions | Use Case                         |
| :------- | :--------- | :------------------------------- |
| `small`  | 32×32px    | Dense lists, tables, inline text |
| `medium` | 48×48px    | Default, comments, cards         |
| `large`  | 64×64px    | Profiles, headers                |
| `xlarge` | 96×96px    | Hero sections, user profiles     |

### Spacing Values

| Spacing | Stack Offset | Use Case                    |
| :------ | :----------- | :-------------------------- |
| `tight` | -12px        | Compact displays            |
| `normal`| -8px         | Default overlapping         |
| `loose` | -4px         | More spacing between items  |

## Accessibility

The AvatarGroup component follows WCAG 2.2 Level AA guidelines:

### Keyboard Navigation

- **Arrow Keys**: Navigate between avatars when interactive (`ArrowRight` / `ArrowLeft`)
- **Tab**: Move focus into and out of the avatar group
- **Enter/Space**: Activate clickable avatars and overflow button

### Screen Reader Support

| Element           | Role     | ARIA Attributes                                      |
| :---------------- | :------- | :--------------------------------------------------- |
| AvatarGroup Root  | `group`  | `aria-label="Avatar group with {count} members"`     |
| Avatar (static)   | `img`    | `aria-label="{username}'s avatar"`                   |
| Avatar (clickable)| `button` | `tabIndex={0}`, `role="button"`                      |
| Overflow Button   | `button` | `aria-label="Show {count} more users"`               |

### Motion Preferences

- All animations use `motion-safe:` prefix
- Respects `prefers-reduced-motion` media query
- Hover expansion effects are disabled for users with reduced motion preferences

### Touch Targets

- Minimum touch target size of **24×24px** (WCAG 2.5.5)
- Overflow button matches avatar size for consistent touch targets

### Best Practices

1. **Provide meaningful `username`**: Enables automatic alt text generation for each avatar
2. **Handle `onOverflowClick`**: Make overflow indicator interactive to reveal hidden users
3. **Use semantic grouping**: The `role="group"` provides context for screen readers
4. **Limit `maxDisplay`**: Keep visible avatars to a reasonable number (5-7) for cognitive accessibility
