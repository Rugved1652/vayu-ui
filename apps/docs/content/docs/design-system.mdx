# Design System

Welcome to the Vayu UI Design System. This page documents the global CSS variables, color tokens, typography, animations, and other foundational design elements used throughout the component library.

## Overview

Vayu UI is built on **Tailwind CSS v4** with a comprehensive design token system. The design system includes:

- **Color System** - Semantic color tokens for light and dark modes
- **Typography** - Fluid type scale with multiple font families
- **Spacing & Shape** - Consistent spacing and border radius
- **Animations** - Ready-to-use keyframe animations
- **Transitions** - Smooth transition timing functions

All tokens are defined in the global CSS file and can be used directly in your components.

## Color System

The color system is organized into semantic color groups with 10 shades each (100-950). Colors are designed to work seamlessly in both light and dark modes.

### Animation Modifiers

Control animation behavior with modifier classes.

| Modifier | Description |
| :--- | :--- |
| `.animate-iteration-1` | Play animation once |
| `.animate-iteration-2` | Play animation twice |
| `.animate-iteration-3` | Play animation three times |
| `.animate-infinite` | Loop animation infinitely |
| `.animate-fill-none` | Don't retain styles after animation |
| `.animate-fill-forwards` | Retain final styles |
| `.animate-fill-backwards` | Apply initial styles before animation |
| `.animate-fill-both` | Apply both initial and final styles |

```tsx
// Animation with modifiers
<div className="animate-bounce-in animate-infinite">
  Infinite bounce
</div>

<div className="animate-fade-in animate-fill-forwards">
  Fade in and stay visible
</div>

<div className="animate-slide-in-up animate-iteration-2">
  Slide up twice
</div>
```

### Reduced Motion

All animations respect the `prefers-reduced-motion` media query for accessibility. Users who have reduced motion enabled will see animations disabled automatically.

```css
/* Automatically handled by CSS */
@media (prefers-reduced-motion: reduce) {
  .animate-* {
    animation: none !important;
  }
}
```

## Dark Mode Inversion Pattern

The design system follows a consistent dark mode inversion pattern.

| Light Mode | Dark Mode |
| :--- | :--- |
| Background → `ground-50/100` | `ground-950/900` |
| Surface → `ground-100/200` | `ground-900/800` |
| Border → `ground-200/300` | `ground-700/800` |
| Text body → `ground-600/700` | `ground-300/400` |
| Text strong → `ground-800/900` | `ground-100/200` |
| Text muted → `ground-400/500` | `ground-500/600` |

### Semantic Color Pairing

For both light and dark modes:

- **[color]-100/200** as background + **[color]-700/800** as text
- **[color]-500** as icon/indicator (works on both bg types)
- **[color]-600** as border + label text

```tsx
// Semantic color pairing example
<div className="bg-primary-100 dark:bg-primary-900/30 text-primary-700 dark:text-primary-300">
  Consistent contrast in both modes
</div>
```

## Global Contrast Rules (WCAG 2.1 AA)

- Never render text below shade **600** on white/light bg
- Never render text above shade **400** on dark/950 bg
- For **primary (lime)**: use **700+** for text — lime is low-contrast

### Installing the Design System

To use Vayu UI's design system in your project:

1. Install the package:
   ```bash
   npm install vayu-ui
   ```

2. Import the global CSS in your app:
   ```tsx
   import 'vayu-ui/global.css';
   ```

3. Start using the design tokens:
   ```tsx
   import { Button } from 'vayu-ui';

   export default function App() {
     return (
       <Button className="bg-primary-500 hover:bg-primary-600">
         Get Started
       </Button>
     );
   }
   ```

## Accessibility

The Vayu UI design system is built with accessibility in mind:

- **WCAG 2.1 AA** compliant color contrast ratios
- **Reduced motion** support for vestibular disorders
- **Semantic** color naming for screen readers
- **Keyboard** navigable components with focus indicators
- **Dark mode** support with proper contrast ratios

## Best Practices

1. **Use semantic colors** instead of arbitrary colors
2. **Respect user preferences** for reduced motion
3. **Maintain contrast ratios** in both light and dark modes
4. **Use type scale utilities** instead of custom font sizes
5. **Leverage transitions** for smooth state changes
6. **Test accessibility** with screen readers and keyboard navigation
