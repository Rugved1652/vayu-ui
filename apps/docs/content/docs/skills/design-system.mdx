# Design System

Guidelines for using Vayu UI's design tokens and composing components consistently.

## Design Tokens

Always use the semantic token system instead of hardcoding values.

### Base Layers

| Token                           | Usage                          |
| ------------------------------- | ------------------------------ |
| `canvas` / `canvas-content`     | App-level backgrounds and text |
| `surface` / `surface-content`   | Container backgrounds and text |
| `elevated` / `elevated-content` | Modals, popovers, dropdowns    |

### Semantic Colors

| Token                     | Usage                               |
| ------------------------- | ----------------------------------- |
| `brand` / `brand-content` | Primary actions, CTAs               |
| `success`                 | Positive feedback, completed states |
| `warning`                 | Caution, pending states             |
| `destructive`             | Errors, destructive actions         |
| `info`                    | Neutral information                 |
| `muted` / `muted-content` | De-emphasized UI                    |

### Structural

| Token    | Usage              |
| -------- | ------------------ |
| `border` | Dividers, outlines |
| `field`  | Input backgrounds  |
| `focus`  | Focus rings        |

### Radius

| Token             | Usage             |
| ----------------- | ----------------- |
| `rounded-control` | Buttons, inputs   |
| `rounded-surface` | Cards, containers |
| `rounded-overlay` | Modals, popovers  |

### Shadows

| Token             | Usage             |
| ----------------- | ----------------- |
| `shadow-control`  | Buttons, inputs   |
| `shadow-surface`  | Cards, list items |
| `shadow-elevated` | Modals, dropdowns |

## Component Composition

Follow the compound component pattern for complex UI:

```tsx
export const Button = Object.assign(ButtonRoot, { Icon, Badge, Text });

// Usage
<Button>
  <Button.Icon>...<Button.Icon>
  <Button.Text>Click me<Button.Text>
<Button>
```

## Anti-Patterns

- **Hardcoding colors** — Always use design tokens
- **Using px values for spacing** — Use the spacing scale
- **Mixing radius values** — Stick to the semantic radius tokens
- **Custom shadows** — Use the predefined shadow tokens
