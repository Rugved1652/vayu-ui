# Folder Structure

Conventions for organizing files and directories in Vayu UI projects.

## Monorepo Structure

```
├── apps/
│   └── docs/              # Documentation site
├── packages/
│   ├── ui/                # Core components and hooks
│   ├── cli/               # vayu-ui CLI
│   ├── registry/          # Component registry metadata
│   └── mcp/               # MCP server
├── scripts/               # Build and automation scripts
└── skills/                # AI skill templates
```

## Component Organization

Each component lives in its own directory:

```
ComponentName/
├── ComponentName.tsx       # Root component
├── ComponentNameChild.tsx  # Sub-components
├── index.ts               # Public exports
├── types.ts               # Component types
└── README.md              # Component documentation
```

## File Naming

- **Components**: PascalCase (`Button.tsx`, `AvatarGroup.tsx`)
- **Hooks**: camelCase with `use` prefix (`useDebounce.ts`)
- **Utils**: camelCase (`formatDate.ts`)
- **Types**: PascalCase with descriptive names (`ButtonProps.ts`)
- **Tests**: Same name as file + `.test.ts` (`Button.test.tsx`)

## Import Organization

Group imports in this order with blank lines between:

1. React / framework imports
2. External library imports
3. Internal absolute imports (`@/components`)
4. Relative imports (`./types`)

```tsx
import React from 'react';

import { clsx } from 'clsx';

import { Button } from '@/components/ui';

import { useCustomHook } from './hooks';
```

## Anti-Patterns

- **Flat file dumps** — Group related files in directories
- **Inconsistent naming** — Stick to one convention per file type
- **Deep nesting** — Keep directory depth to 3 levels max
- **Circular imports** — Structure code to avoid dependency cycles
